About Typescene

Typescene is a strongly typed front-end GUI toolkit for modern Web applications.

It can be used to develop a User Interface layer for applications that run in the browser (or a browser-like environment, such as Electron).

For updates on Typescene developments, follow @typescene on twitter.

Who made this?

Typescene is an open source project, accepting issue reports and contributions from the community on GitHub.

The Typescene project was started by Jelmer Cormont, independent web developer and consultant. It is the result of many years of experimentation, development and refactoring, in search of the perfect balance between convenience and consistency; something to power small projects hassle-free with a drop-in library file, while also supporting 'enterprise' systems with a more complex architecture.

Why another framework?

There are a few things that set Typescene apart from the majority of other JavaScript frameworks on the Web today.

  • Most importantly, Typescene allows you to write an application entirely in TypeScript, a strongly typed superset of JavaScript developed by Microsoft. While the browser still renders HTML (through the DOM) and CSS, the details of how these are generated are encapsulated in the core UI code. This makes large projects much more manageable, as a set of .ts files organized by component or feature.
  • Typescene provides an API that resembles desktop GUI toolkits more than other Web frameworks. The structure of this API, with component objects and factories, is easily understood, instantly familiar to developers who also create desktop apps, and lends itself well to a clean object oriented architecture.
  • A coherent set of asynchronous programming constructs, woven into the fabric of Typescene’s APIs, makes for highly readable asynchronous code. Components can be rendered asynchronously, content (state) observed and updated automatically, events received and signals sent to parent components or activities, all with only a few simple constructs provided by the toolkit.
  • Unlike some newer JavaScript frameworks, Typescene still supports Internet Explorer 9, 10, and 11, which may be a requirement for business applications. The toolkit itself has no further dependencies (other than the TypeScript compiler), so code written today can be guaranteed to compile and run years from now.

How do I contribute?

You can submit issue reports and feature requests on the GitHub issues page.

Pull requests are welcome. Do adhere to the coding style observed in the rest of the source code, and know that new features may not be implemented until a full testing framework has been put into place.

When in doubt, tweet @typescene or open an issue just to check if your idea is worthwhile, and whether anyone else is working on the same thing.

Documentation is generated automatically from a mixture of source code comments and Markdown files in the src/doc folder of each module. Additional documentation and further samples should also be worked on.
